Setup · 1.1

🔑 본사 관리자 로그인

2가지 방법: ① 본인 경영팀 계정 로그인 ② "본사 관리자" 카드 → 관리자 비밀번호
방법 1 · 경영팀 개인 계정
1
로그인 카드 → 전화번호 + 비번
2
자동으로 HQ 대시보드 진입
role=경영팀 또는 brand=DM INTERNATIONAL이면 HQ 권한 자동 부여
방법 2 · 마스터 관리자
1
시작 화면 → "본사 관리자" 카드
2
관리자 비밀번호 입력
기본값: 01082470187
3
"본사 관리자" 권한으로 진입
phone=admin · 모든 권한
!
마스터 비밀번호는 노출되면 위험합니다. 가능하면 본인 경영팀 계정으로 로그인하세요. 마스터는 비상시 백업용.
✓
두 방법 모두 동일한 권한 (직원 관리·발주 처리·아카데미 등 모두 가능). 마스터는 휴게/근태 기록이 본인 데이터가 아니라 "admin"으로 들어가므로 주의.

Staff · 2.1

👤 가입 승인 처리

사이드바 → "가입 승인". 신규 직원이 가입 신청한 내용을 검토하고 승인.
1
신청자 정보 확인
이름, 활동명, 생년월일, 전화번호, 브랜드, 지점, 직급, 레벨/주차
2
정보가 정확한지 본인에게 확인
특히 활동명·생년월일·입사일은 추후 변경이 번거로움
3
"승인" 버튼 클릭 → 모달 확인
4
자동 권한 부여
디자이너 → academyRole=교육자 / 인턴 → academyRole=인턴 자동 설정
5
신청 정보를 직원 DB로 이전 + pending에서 삭제
!
승인 직후 발주 권한은 OFF 기본값. 발주 권한 줘야 하면 직원 관리에서 별도 변경.

🔄 담당 디자이너 자동 복구

디자이너↔인턴 연결이 사라졌을 때 교육 이력 기반으로 자동 복원. 50명 본격 사용 전 1회 필수 실행 권장.
!
담당 미지정 인턴이 있으면 디자이너 대시보드의 "내 인턴" 위젯이 비어있게 됩니다. 빨리 복구해주세요.
1
사이드바 → "아카데미 관리"
2
"담당 미지정 N명" 카드 확인
0명이면 복구 불필요
3
"🔄 담당 디자이너 자동 복구" 버튼 클릭
4
모달에서 추정 결과 검토
교육 이력 기반 가장 최근 교육자 추정 + 최다 교육자 참고 표시
5
체크박스 확인 → "선택 항목 일괄 저장"
담당 미지정만 자동 체크. 기존 담당자 변경은 수동 체크 필요
6
확인 모달 → 일괄 적용
✓
복구 후 디자이너 대시보드/직원 관리에 즉시 반영됩니다.

📦 발주 관리 (대기 → 배송완료)

사이드바 → "발주 관리". 모든 지점에서 신청한 발주를 처리.
발주 상태 흐름
⏳ 대기→📦 확인→✅ 배송완료
✅ 배송완료→🔄 환불요청→💰 환불완료
⏳ 대기→❌ 발주 취소 (재고 없음 등)
1
기간/브랜드/지점/상태 필터로 좁히기
2
탭으로 상태별 분리 보기
전체 / 대기 / 확인 / 배송완료 / 환불요청 / 환불완료 / 발주 취소
3
발주 카드별 액션 버튼
상태에 따라 "확인" / "배송완료" / "환불 처리" / "발주 취소" 버튼 표시
4
📥 엑셀 내보내기 가능
!
"발주 취소"는 신청을 거절할 때. "환불"은 이미 배송된 후 문제 있을 때. 두 개 헷갈리지 마세요.
i
발주 카드의 "📞 연락처" 버튼으로 신청자에게 직접 통화/문자 가능 (모바일).

사이드바 → "발주 통계". 제품/브랜드/지점/직원별 발주 분석. 재고 관리·예산 책정에 활용.

·

🎓 아카데미 관리 (인턴 카드 클릭)

사이드바 → "아카데미 관리". 모든 인턴의 진도 관리 + 주차 직접 체크 + 담당 디자이너 변경 통합.
1
상단 카드 (전체 인턴 / 담당 미지정 N명)
2
레벨별 분포
3
인턴 현황 카드 (레벨순 정렬)
활동명·레벨/주차·담당 디자이너·진도율
4
인턴 카드 클릭 → 모달 열림
담당 디자이너 변경 + 주차별 체크 + 옵션으로 교육자/날짜 입력 시 교육 이력에도 등록
5
주차 grid 클릭으로 토글
완료 ↔ 미완료. 비순차 체크 가능 (예: 14주차만 체크)
6
"저장" → 즉시 인턴 진도에 반영
✓
디자이너가 교육 등록 깜빡한 케이스도 본사가 여기서 직접 체크 가능. 교육자 입력하면 교육 이력에도 자동 기록.
i
"교육자 선택 안 함 (수동 체크만)" 고르면 진도만 수정되고 교육비 산정엔 반영 안 됨.
!
담당 디자이너가 없는 인턴은 카드에 ⚠️ 노란 뱃지 표시. 모달에서 즉시 변경 가능.

FAQ · 6.1

❓ 자주 묻는 질문

Q.50명 동시 사용 시 끊김 없을까요?
Firebase Spark(무료)는 동시 접속 100개 한계. 50명이면 평균 60~80개 예상 → 피크 시 일시 거부 가능. Blaze(종량제, 월 1만원 이하)로 전환하면 200,000개로 상향.
Q.두 관리자가 동시에 같은 발주를 처리하면?
Last-write-wins (늦게 저장한 사람이 덮어씀). 데이터 손실은 없지만 상태가 의도와 다를 수 있음. 본사 작업은 한 명 담당 권장.
Q.옛 PWA에서 새 기능이 안 보여요
Service Worker 도입 전에 설치한 PWA는 캐시 문제. 직원에게 PWA 재설치 가이드 PDF 발송. 다음부턴 자동 업데이트.
Q.잘못 거절한 발주를 되살리고 싶어요
발주 관리 → "발주 취소" 탭 → 해당 발주 → 상태 변경. 직접 DB 수정도 가능 (Firebase Console).
Q.퇴사 직원의 데이터는 어떻게 되나요?
"퇴사" 처리 시 status=퇴사로만 변경되고 데이터 보존. "영구 삭제"는 신중히. 발주 통계나 교육 이력에서 이름은 남음.
