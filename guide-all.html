CHAPTER B · 인턴

인턴 핵심 기능

자세한 내용은 별도 인턴 가이드 참고
Intern · B.1

인턴이 매일 쓰는 핵심 3가지

시스템에서 인턴이 가장 많이 사용하는 기능들. 각 기능의 자세한 사용법은 인턴 가이드 PDF 참고.
📈
내 진도 (주차 grid)
사이드바 → "내 진도". 1~16주차를 색상으로 구분: 완료(녹색), 현재(파랑), 미수강(회색). 비순차 수강도 정확히 표시.
⏱️
연습 측정 + 랭킹
사이드바 → "연습 측정". 종목 선택 → 시작 → 종료. 자동으로 시간 기록되어 전체 인턴 랭킹에 반영.
📚
외부 수강 등록
외부 아카데미 다녀왔으면 "외부 수강"에서 직접 입력. 날짜 + 수강 주차 (1~16, 비순차 OK). 즉시 진도 반영.
📝
시험 결과·숙제
시험 결과는 본사가 입력 → "시험 결과"에서 확인. 숙제는 디자이너가 출제 → "숙제"에서 확인 후 완료 체크.
📦
발주하기 (권한 시)
권한 ON이면 사이드바에 "발주하기" 표시. 카테고리 → 제품 → 수량 → 카트 → 제출.
🌴
휴가 신청
잔여 연차 확인 → 날짜 선택 → 사유 입력 → 본사 승인 대기. 보유 + 추가 - 사용 = 잔여.

CHAPTER C · 디자이너

디자이너 핵심 기능

자세한 내용은 별도 디자이너 가이드 참고
Designer · C.1

디자이너가 매일 쓰는 핵심 기능

담당 인턴의 진도를 책임지고 관리하는 디자이너의 주요 기능. 본사 교육비 정산의 기초가 되므로 정확한 입력이 중요합니다.
👥
내 인턴 관리
사이드바 → "내 인턴". 담당 인턴 전체 진도·잔여 연차·지각 누적·실시간 휴게 상태 한눈에.
✅
교육 완료 등록
사이드바 → "교육 완료". 날짜·레벨·주차 선택 → 수강생 다중 체크. 1회당 ₩60,000 자동 산정.
📝
숙제 출제
사이드바 → "숙제 출제". 인턴 선택 → 마감 + 내용. 다음 교육 등록 시 이전 숙제 자동 완료.
📊
교육 실적 보기
사이드바 → "교육 실적". 이번달 교육 횟수·교육 인턴 수.
☕
지점 휴게 현황
본인 지점 인턴들이 휴게 중·외근·완료 상태 실시간 모니터링.
🌴
휴무 신청
사이드바 → "휴무 신청". 인턴의 "연차"와 다른 카테고리. 잔여일 차감 X.

📦
발주 현황
사이드바 → "발주 현황". 본인 지점 발주 상태 (대기/확인/배송/취소/환불) 한눈에. 기간/탭 필터.
📦
발주하기 (권한 시)
권한 ON이면 사이드바에 "발주하기" 표시. 인턴과 동일한 절차.
📋
발주 내역
권한 시 본인이 신청한 발주 진행 상태 확인. 환불 요청도 여기서.
🌴
휴무 신청
사이드바 → "휴무 신청". 본사 관리자 승인 후 적용.

경영팀 핵심 기능

HQ · E.1

경영팀이 다루는 핵심 영역 6개

전사 운영을 책임지는 본사 관리자의 핵심 책임 영역. 자세한 절차는 별도 28장 가이드 참고.

직원 관리

50명 본격 사용 전 1회 필수: 사이드바 → 아카데미 관리 → "🔄 담당 디자이너 자동 복구" 실행

FAQ

❓ 모두를 위한 자주 묻는 질문

Q.비밀번호를 잊었어요
로그인 화면 → "비밀번호 재설정" → 이름·전화번호·생년월일 입력 → 본사 관리자가 새 비번 발급. 또는 본사 관리자에게 직접 연락이 더 빠릅니다.
Q.새 기능이 안 보여요
옛 PWA 캐시 문제. 위 슬라이드 19 참고해서 재설치. 다음부턴 자동 업데이트.
Q.주차 표시가 이상해요 (15주차인데 7/16으로 나옴)
"수강한 주차 / 총 주차" 형식. 7/16은 16주차 중 7주차를 받았다는 뜻 (꼭 1~7일 필요는 없고 비순차 가능).
Q.발주하기 메뉴가 안 보여요
발주 권한이 OFF 상태. 본사 관리자에게 권한 요청.
Q.담당 인턴/디자이너가 사라졌어요
본사 관리자에게 "아카데미 관리" → "🔄 담당 디자이너 자동 복구" 요청. 교육 이력 기반으로 자동 복원됨.
Q.휴게 60분 초과하면?
자동 정지 X. 본사 통계에 "휴게초과"로 카운트. 직원 관리에서 이번달/올해 누적 표시.
Q.실시간 동기화는 얼마나 잘 되나요?
Firebase Realtime Database 기반. 1~2초 내 모든 사용자 화면에 반영. 백그라운드 시 일시 중단되며, 앱 다시 열면 즉시 동기화.
